# App Store Launch Feature Test Matrix

Purpose: single launch-coverage source for App Store / TestFlight readiness. Use this matrix before every release candidate and update the **Launch decision** after executing the listed automated and manual checks against the candidate build.

Decision legend:

- **Pass** — automated coverage exists and required manual mobile checks passed for the candidate.
- **Blocked** — cannot launch until a missing service, env var, account setting, or critical defect is resolved.
- **Accepted Risk** — known limitation is documented and explicitly approved for this launch.
- **Needs Fix** — repo work is required before release, but external setup is not the primary blocker.

Baseline release commands: `npm run lint:check`, `npm run typecheck`, `npm run test:run`, `npm run build`, `npm run qa:guardrails`, `npm run validate-env`, and the relevant Playwright specs below.
